HostingLint

Static analysis for hosting control panel modules: WHMCS (PHP), cPanel (Perl), and OpenPanel (Docker). No PHP or Perl runtime required—safe for CI and local checks.

Packages

hostinglint — Command-line interface (depends on core).

@hostinglint/core — Analysis engine and public TypeScript API.

Both are MIT-licensed and require Node.js 20+.

Install

CLI (no install)

npx hostinglint check ./path/to/module/

CLI (global)

npm install -g hostinglint
hostinglint check ./path/to/module/

Library

npm install @hostinglint/core

CLI: hostinglint check

Analyze a file or directory. Supported extensions are detected automatically (e.g. .php, .pl, Dockerfile, OpenPanel manifests).

OptionDescription
<path>File or directory to analyze (required).
-p, --platformwhmcs, cpanel, or openpanel. Omit for auto-detection.
--php-versionTarget PHP version for compatibility rules (default 8.3).
-f, --formattext (default), json, or sarif (GitHub Code Scanning).
-c, --configPath to .hostinglintrc.json (otherwise searched upward from target).
-w, --watchRe-run when files change (where supported).
--fixApply automatic fixes where the engine supports them, then re-check.
--no-securityDisable security rules.
--no-best-practicesDisable best-practice rules.

Exit code is non-zero when any error-severity finding remains (single pass mode).

Library API (@hostinglint/core)

Typical entry points:

  • analyzeAuto(code, filePath, options) — pick analyzer by file path.
  • analyzePhp, analyzePerl, analyzeOpenPanel — platform-specific analysis.
  • findConfig, loadConfigFromFile, mergeConfig — configuration.
  • applyFixes, getFixableSummary — fix pipeline.
  • Rule registry: allRules, getRuleById, presets recommended, strict, etc.

Types (e.g. LintResult, Platform, PhpVersion) ship with the package. Use Node’s Node16/NodeNext resolution and .js extensions in relative imports when mirroring the library’s style.

import { analyzePhp, analyzeAuto, findConfig } from "@hostinglint/core";

const results = analyzePhp(source, "module.php", {
  phpVersion: "8.3",
  whmcsVersion: "8.13",
  security: true,
});

Configuration

Optional project file .hostinglintrc.json:

  • rules — per-rule severity or off.
  • phpVersion, WHMCS/cPanel version fields as supported by core.
  • ignore — glob patterns for paths to skip.

Inline suppressions in source comments (e.g. hostinglint-disable-next-line rule-id) are supported for PHP/Perl-style comments—see the GitHub README for exact syntax.
